
# 🔍 Root Cause Analyzer Metrics

Welcome to **Root Cause Analyzer Metrics** — a high-performance system designed to **capture, process, and analyze operational metrics in real time** so you can quickly understand **what’s happening in your system and why**.

Built with **TypeScript** and a **serverless cloud architecture**, this project makes it easy to collect performance signals, process them asynchronously, and surface insights that help identify potential bottlenecks before they become real problems.

Whether you're debugging performance issues, building observability pipelines, or experimenting with scalable telemetry collection, this project provides a **lightweight and extensible foundation**.

---

# 🚀 What This Project Does

Root Cause Analyzer Metrics acts as a **metrics ingestion and processing pipeline**.

Applications send metric events to an API endpoint. Those events are then:

1. **Accepted by serverless compute functions**
2. **Queued for asynchronous processing**
3. **Processed and emitted as structured metrics**
4. **Logged and monitored for real-time analysis**

This design ensures the system stays **fast, reliable, and scalable**, even under heavy traffic.

---

# 🏗 High-Level Architecture

The system is composed of three core components:

### ⚡ Serverless Compute

Handles incoming requests and performs metric processing logic. These functions automatically scale based on incoming traffic.

### 📊 Cloud Monitoring

Captures logs and metrics generated by the system, enabling real-time visibility into application behavior and performance trends.

### 📬 Distributed Message Queue

Incoming events are placed into a queue and processed asynchronously. This guarantees **reliability, fault tolerance, and smooth handling of spikes in traffic**.

# 🧪 Try It Locally

You can get the project running in just a few minutes.

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd root-cause-analyzer-metrics
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Build the Project

Compile the TypeScript source:

```bash
npm run build
```

This will generate compiled output in the `/dist` directory.

---

### 4. Package the Function

Create a deployment bundle that includes:

* compiled `/dist` files
* `node_modules`
* project configuration files

Example:

```bash
zip -r deployment.zip dist node_modules package.json
```

---

### 5. Deploy to Your Cloud Environment

Upload the deployment bundle to your serverless function runtime and configure an HTTP endpoint that triggers the metrics collector.

Once deployed, you will have an endpoint that looks like:

```
https://<your-endpoint>/prod/collect
```

---

### 6. Send a Test Request

You can simulate a metrics event using:

```bash
awscurl --service execute-api \
-X POST https://<your-endpoint>/prod/collect
```

Or with curl:

```bash
curl -X POST https://<your-endpoint>/prod/collect \
-H "Content-Type: application/json" \
-d '{
  "service": "checkout",
  "event": "latency",
  "value": 142
}'
```

You should then see the event flow through the pipeline and appear in the system logs and metrics stream.
